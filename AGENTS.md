# AGENTS.md

## Project Overview

**sv-router** is a type-safe routing library for Svelte 5 single-page applications. It supports both code-based and file-based routing, with features like nested layouts, dynamic routes, preloading, hooks, and reactive search params.

This is a **pnpm monorepo** with the following packages:

- `/`: The main `sv-router` library
- `create/`: `create-sv-router` scaffolding CLI. Templates live in `create/templates/`.
- `docs/`: VitePress documentation site. Guide pages are in `docs/guide/code-based/` and `docs/guide/file-based/`.
- `examples/*`: Example apps

## Usage

### Code-Based Routing

Routes are defined as a nested object passed to `createRouter()`, which returns typed helpers:

```js
const { p, navigate, isActive, preload, route } = createRouter({
	'/': Home,
	'/about': About,
	'/posts': {
		'/': Posts,
		'/:slug': PostDetail, // dynamic segment
		layout: PostsLayout, // wraps all routes at this level
		hooks: { beforeLoad }, // route guards / data loading
		meta: { requiresAuth: true },
	},
	'*notfound': NotFound, // catch-all
});
```

Type safety is fully inferred from the route definition (no code generation needed). `p()`, `navigate()`, and `isActive()` all validate paths and params at compile-time. Routes can be lazy-loaded with `() => import('./Route.svelte')`.

### File-Based Routing

Routes are auto-generated from `src/routes/` via a Vite plugin. File conventions:

- `index.svelte` → `/`, `about.svelte` → `/about`
- `[id].svelte` → `/:id` (dynamic), `[...catch].svelte` → catch-all
- `layout.svelte` → wraps sibling/child routes
- `hooks.ts` → route hooks for that directory
- `(groupName)/` → layout groups (not in URL)

Typed helpers are imported from `sv-router/generated` (generated by the Vite plugin).

### Key APIs

- **`p(path, opts?)`**: build a type-safe URL with params/search/hash
- **`navigate(path, opts?)`** or **`navigate(-1)`**: programmatic navigation
- **`route`**: reactive state: `.pathname`, `.params`, `.search`, `.hash`, `.meta`, `.getParams(path)`
- **`searchParams`**: reactive URLSearchParams wrapper (`.set()`, `.get()`, `.delete()`, auto-updates URL)
- **`blockNavigation(callback)`**: prevent navigation (e.g. unsaved changes)
- **`isActiveLink({ className, startsWith? })`**: Svelte 5 attachment for active link styling
- **`Router`** component: renders the matched route, accepts optional `base` prop

### Hooks

Defined per route level, run in order from root to leaf:

- **`beforeLoad`**: async guard, can `throw new Navigation('/redirect')` to redirect
- **`afterLoad`**: runs after route renders
- **`onPreload`**: runs when route is preloaded
- **`onError`**: catches component load or hook errors

### Preloading

Add `data-preload` to `<a>` tags. Strategies: hover (default), `"viewport"`, `"predict"` (cone-based pointer prediction). Also available programmatically via `preload()`.

### Testing

- **Framework:** Vitest with happy-dom environment
- **Libraries:** @testing-library/svelte, @testing-library/user-event, @testing-library/jest-dom
- **Setup file:** `tests/vitest-setup.js`
- **Test location:** `tests/` directory, files named `*.test.js`

### Code Style

- **Source language:** JavaScript with JSDoc type annotations (not `.ts`). Public API types are in `src/index.d.ts`.
- **Formatting:** Prettier tabs, single quotes, 100 char width
- **Imports:** Sorted into a single group
- **Filenames:** kebab-case (except `.svelte` files)
- **No console.log:** Only `console.warn` is allowed
- **Svelte 5:** Runes mode enabled

### Release

Uses **Changesets** for version management. CI (GitHub Actions) runs lint, type check, and tests on push/PR.
